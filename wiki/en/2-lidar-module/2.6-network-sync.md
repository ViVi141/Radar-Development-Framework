# LiDAR Network Sync

The LiDAR network module provides server-authoritative scan sync for multiplayer.

---

## Server-Authoritative Model

- **Server**: Performs scan, serializes to CSV, broadcasts via RPC.
- **Client**: Receives scan data, parses CSV, can render or process.

---

## Components

| Class | Role |
|-------|------|
| `RDF_LidarNetworkAPI` | Base API interface (empty default) |
| `RDF_LidarNetworkComponent` | Rpl-based implementation; RplProp sync + RPC |
| `RDF_LidarNetworkUtils` | Find API on entity, bind AutoRunner |
| `RDF_LidarNetworkScanner` | Network-aware scanner (sync or local fallback) |

---

## RDF_LidarNetworkComponent

Attach to an entity (e.g. vehicle). Synchronizes:

- `m_DemoEnabled`, `m_RayCount`, `m_UpdateInterval`, `m_RenderWorld`, `m_DrawOriginAxis`, `m_Verbose`

Scan results: Server serializes via CSV and broadcasts with `RpcDo_ScanCompleteWithPayload(string csv)`.

---

## Binding AutoRunner

```c
RDF_LidarNetworkUtils.BindAutoRunnerToLocalSubject(true);
RDF_LidarAutoRunner.SetNetworkAPI(RDF_LidarNetworkUtils.FindNetworkAPI(entity));
```
