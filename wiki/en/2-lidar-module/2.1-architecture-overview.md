# LiDAR Architecture Overview

The LiDAR module provides laser-based point-cloud scanning, visualization, and demos. It is the base layer that the Radar module extends.

---

## Design Principles

- **Opt-in**: Runs only when explicitly enabled; does not affect other mods.
- **Modularity**: Scanning core, physics, visualization, and demo logic are isolated and can be replaced independently.
- **Extensibility**: Stable extension interfaces (sample strategies, color strategies, scan-complete callbacks).

---

## Data Flow

```
RDF_LidarSettings
       |
       v
RDF_LidarScanner.Scan(entity)
  +-- SampleStrategy.BuildDirection() x N
  +-- Trace(ray) -> RDF_LidarSample
  +-- Returns array<ref RDF_LidarSample>
       |
       v
RDF_LidarVisualizer.RenderWithSamples()  (optional: 3D point cloud/rays)
       |
       v
RDF_LidarScanCompleteHandler.OnScanComplete()  (optional callback)
```

---

## Module Layout

| Directory | Role |
|-----------|------|
| **Core/** | Settings, types, scanner, sample strategies |
| **Visual/** | Point cloud/ray rendering, color strategies |
| **Util/** | Subject resolver, CSV export, stats/filter, callbacks |
| **Demo/** | AutoRunner, DemoConfig, Cycler, Bootstrap |
| **Network/** | Server-authoritative sync (Rpl) |
| **UI/** | RDF_LidarHUD (PPI point-cloud view) |

---

## Inheritance Relationship

`RDF_RadarScanner` extends `RDF_LidarScanner` and reuses the ray-trace flow. Radar adds `ApplyRadarPhysics()` on each hit, but the LiDAR module itself has no dependency on radar.
