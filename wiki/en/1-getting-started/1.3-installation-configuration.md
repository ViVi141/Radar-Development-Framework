# Installation & Configuration

## Installation

1. Add the **Radar Development Framework** addon to your Arma Reforger Workbench project.
2. Ensure the addon is enabled in your scenario or game mode.
3. The framework is **opt-in**: it does nothing until explicitly enabled via Bootstrap or programmatic API.

---

## Addon Configuration

- **Project ID**: `RadarDevelopmentFramework`
- **addon.gproj**: Standard Arma Reforger addon project descriptor.
- **Dependencies**: See `addon.gproj` for required engine modules.

---

## Bootstrap Configuration

### LiDAR Bootstrap (`RDF_LidarAutoBootstrap.c`)

```c
SCR_BaseGameMode.SetBootstrapEnabled(true);           // Enable LiDAR demo at game start
SCR_BaseGameMode.SetBootstrapAutoCycle(true);         // Enable strategy auto-cycle
SCR_BaseGameMode.SetBootstrapAutoCycleInterval(10.0); // Cycle interval in seconds
```

### Radar Bootstrap (`RDF_RadarAutoBootstrap.c`)

```c
SCR_BaseGameMode.SetRadarBootstrapEnabled(true);      // Enable radar demo at game start
SCR_BaseGameMode.SetRadarBootstrapAutoCycle(true);    // Enable preset auto-cycle
SCR_BaseGameMode.SetRadarBootstrapShowHUD(true);      // Show HUD (default)
SCR_BaseGameMode.SetRadarBootstrapShowAScope(false);  // Show A-Scope (optional)
SCR_BaseGameMode.SetRadarBootstrapShowVoxel(false);   // Show voxel debug (optional)
```

---

## Configuration via Source

You can also modify static variables directly in the bootstrap files:

**RDF_LidarAutoBootstrap.c**:

- `s_BootstrapEnabled` — Enable LiDAR demo at startup (default: false)
- `s_BootstrapAutoCycle` — Auto-cycle strategies (default: false)
- `s_BootstrapAutoCycleInterval` — Interval in seconds

**RDF_RadarAutoBootstrap.c**:

- `s_RadarBootstrapEnabled` — Enable radar demo at startup (default: false)
- `s_RadarBootstrapShowHUD` — Show HUD (default: true)
- `s_RadarBootstrapConfigIdx` — Default preset index (0–5)
