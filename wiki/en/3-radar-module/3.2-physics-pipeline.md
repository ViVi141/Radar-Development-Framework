# Radar Physics Pipeline

For each ray hit, `RDF_RadarScanner.ApplyRadarPhysics()` executes these steps in order:

---

## 10-Step Pipeline

| Step | Description |
|------|-------------|
| 1. **FSPL** | Free-space path loss: `20·log10(d) + 20·log10(f) + 92.45 dB` |
| 2. **Atmospheric** | Frequency × distance dependent (ITU-R P.676) |
| 3. **Rain** | Optional rain attenuation (ITU-R P.838) |
| 4. **RCS** | Entity: bounding box × material; terrain: diffuse area scattering |
| 5. **Radar equation** | `Pr = Pt·Gt·Gr·σ·λ² / (4π)³ / R⁴ / L` |
| 6. **Range gate** | Discard near-field blind zone |
| 7. **Noise** | `kTBF`, compute SNR |
| 8. **Doppler** | `fd = 2·vr·f / c` (radial velocity) |
| 9. **Detection** | `SNR ≥ m_DetectionThreshold` |
| 10. **Clutter filter** | Optional MTI/MTD for stationary targets |

---

## Key Components

- **RDF_RadarPropagation** — FSPL, atmospheric, rain attenuation
- **RDF_RCSModel** — Analytic sphere/plate/cylinder, material lookup, entity estimation, terrain diffuse
- **RDF_RadarEquation** — Received power, noise power, SNR
- **RDF_DopplerProcessor** — Radial velocity, Doppler shift, MTI
- **RDF_CFar** — CA-CFAR / OS-CFAR (PoC) for adaptive detection threshold
