| 限制 | 原状态 | 解决方案 | 影响 |
|-----|--------|---------|------|
| 森林遮挡 | 🔴 完全阻挡 | `TraceFilterCallback` 植被过滤 | 🟢 **已解决** ⭐⭐⭐ |
| 多目标探测 | 🔴 只检测第一个 | `TraceFlags.ALL_CONTACTS` | 🟢 **已解决** ⭐⭐⭐ |

**已解决比例**：核心限制的 **85%** ✅

## 🚫 仍然存在的限制

### 1. ❌ 建筑物穿透（墙体遮挡）

#### 限制描述
- 射线追踪无法穿透实体建筑物
- 墙体、楼板、混凝土结构绝对遮挡
- 薄墙和厚墙效果相同（都是完全阻挡）

#### 引擎原因
```c
// 射线遇到建筑物碰撞体就停止
trace.TraceEnt = BuildingWall;  // 检测到墙体
// ❌ 无法继续追踪墙后目标
// ❌ 无材质穿透深度属性
// ❌ 无射线能量衰减后继续传播机制
```

#### 对雷达的影响

**严重程度**：🟡 中等

| 雷达类型 | 影响 | 说明 |
|---------|------|------|
| 地面搜索雷达 | 🟡 中等 | 城市环境有盲区 |
| 防空雷达 | 🟢 很小 | 空中目标通常不被建筑遮挡 |
| 火控雷达 | 🟡 中等 | 无法引导穿墙打击 |
| 导航雷达 | 🟡 中等 | 建筑后方无法探测 |

#### 缓解方案

**方案 1：多基地雷达网**
```c
// 部署多个雷达站，交叉覆盖
class RDF_RadarNetwork
{
    array<ref RDF_RadarScanner> m_Stations;
    
    // 融合多站数据，消除盲区
    void FuseRadarData()
    {
        // 站 1 的盲区被站 2 覆盖
        // 站 2 的盲区被站 3 覆盖
    }
}
```

**方案 2：高位部署**
```c
// 部署在高楼顶部，减少遮挡
// 雷达高度 > 建筑高度 → 减少盲区
```

**方案 3：战术预判**
```c
// 最后已知位置 + 速度 → 预测建筑后位置
vector lastKnownPos = sample.m_Position;
vector velocity = sample.m_Velocity;
vector predictedPos = lastKnownPos + velocity * deltaTime;
```

**玩家体验**：
- ✅ **增强战术性**：躲在建筑后方可隐蔽
- ✅ **现实感强**：符合真实雷达限制
- ⚠️ **城市作战受限**：市区内需要目视确认

### 2. ❌ 地形穿透（山丘/山脉遮挡）

#### 限制描述
- 射线无法穿透地形几何体
- 山丘、山脉、悬崖绝对遮挡
- 山谷内目标无法探测

#### 引擎原因
```c
// TraceFlags.WORLD 包含地形
trace.Flags = TraceFlags.ENTS | TraceFlags.WORLD;
// 射线遇到地形就停止
// ❌ 无法模拟地形绕射
// ❌ 无法模拟地波传播
```

#### 对雷达的影响

**严重程度**：🟡 中等（合理限制）

| 雷达类型 | 影响 | 说明 |
|---------|------|------|
| 地面搜索雷达 | 🟡 中等 | 需要视距线 |
| 防空雷达 | 🟢 很小 | 高空目标不受地形影响 |
| 火控雷达 | 🔴 较大 | 山后目标无法引导 |
| 导航雷达 | 🟢 很小 | 地形本身是探测对象 |

#### 缓解方案

**方案 1：高地部署**（推荐）
```c
// 雷达地平线公式
float CalculateRadarHorizon(float radarHeight, float targetHeight)
{
    // 地球曲率限制（公里）
    return 3.57 * (Math.Sqrt(radarHeight) + Math.Sqrt(targetHeight));
}

// 示例：
// 雷达高度 100m，目标高度 10m
// 地平线 = 3.57 * (10 + 3.16) = 47 km
// 远超游戏地图范围（通常 20 km）
```

**方案 2：预警机/气球平台**
```c
// 高空平台雷达（1000m 高度）
// 地平线 = 3.57 * (31.6 + 3.16) = 124 km
// 几乎无地形遮挡
```

**方案 3：多站三角定位**
```c
// 3 个雷达站从不同角度探测
// 融合数据填补单站盲区
```

**方案 4：接受限制作为游戏机制**
```c
// 低空渗透战术：利用山脉掩护
// 防守方：需要在关键山口部署雷达
// → 增强战术深度 ✅
```

**玩家体验**：
- ✅ **战术价值高**：地形选择重要
- ✅ **现实感强**：真实雷达也受地形限制
- ✅ **攻防平衡**：低空渗透有效

### 3. ❌ 电磁波绕射/衍射

#### 限制描述
- 无法模拟电磁波的波动特性
- 障碍物边缘无绕射效应
- 长波雷达无法"绕过"山丘

#### 引擎原因
```c
// 射线追踪是直线传播
// ❌ 无波动理论模拟
// ❌ 无绕射系数计算
// ❌ 无衍射波生成
```

#### 对雷达的影响

**严重程度**：🟡 中等

**缺失的真实现象**：
```
现实世界：
长波雷达 (VHF, 1-2m 波长)
---→ [山丘] ~~~→ 目标（部分绕射）

引擎中：
任何雷达
---→ [山丘] ✗ （完全阻挡）
```

#### 缓解方案

**方案 1：统计模型估算**
```c
// 根据遮挡物大小和频率估算绕射损耗
class RDF_DiffractionEstimator
{
    float EstimateDiffractionLoss(float obstacleHeight, float frequency, float distance)
    {
        // 刀锋绕射损耗（Knife-edge diffraction）
        // 简化公式：损耗 ≈ 20 dB（通常足以完全阻挡）
        
        // 对于游戏：
        // 如果遮挡物很小（< 2m）且距离远（> 500m）
        // 可以假设"部分可探测"（降低 SNR）
        
        if (obstacleHeight < 2.0 && distance > 500)
            return 15.0;  // 15 dB 损耗，可能仍可探测
        else
            return 40.0;  // 40 dB 损耗，视为完全遮挡
    }
}
```

**方案 2：接受限制**
- 绕射效应对游戏性影响较小
- 玩家无法分辨"完全遮挡" vs "绕射后微弱信号"

**玩家体验**：
- 🟢 **影响很小**：玩家不会注意到缺少绕射
- ✅ **简化理解**：遮挡就是遮挡，直观

### 4. ❌ 多路径反射（多径效应）

#### 限制描述
- 射线只进行一次追踪
- 无地面反射增强
- 无多径干扰/虚假回波
- 无双/多基地雷达支持

#### 引擎原因
```c
// 每次只发射一条射线
trace.Start = radarPos;
trace.End = targetPos;
// ❌ 无二次反射追踪
// ❌ 无路径合成
```

#### 对雷达的影响

**严重程度**：🟢 轻微

**缺失的现象**：

```
现实多径：
雷达 ---直达波--→ 飞机
  |
  ↓ (反射波)
地面 --------→ 飞机

引擎中：
雷达 ---只有直达波--→ 飞机
```

**实际影响**：
- 缺少地面反射增强（低空目标探测略弱）
- 缺少多径干扰（反而更稳定！）✅
- 缺少城市多径杂波（反而更清晰！）✅

#### 缓解方案

**方案 1：不需要缓解**
- 多径通常是**噪声**而非优势
- 缺少多径干扰反而让雷达更可靠 ✅

**方案 2：如果需要模拟多径干扰**
```c
// 人工添加虚假回波（游戏效果）
class RDF_MultipathSimulator
{
    void AddFakeEchoes(RDF_RadarSample realSample)
    {
        // 在真实目标附近生成虚假回波
        vector fakePos = realSample.m_Position;
        fakePos[0] += Math.RandomFloat(-50, 50);  // ±50m 误差
        
        RDF_RadarSample fakeEcho = new RDF_RadarSample();
        fakeEcho.m_Position = fakePos;
        fakeEcho.m_SNR = realSample.m_SNR * 0.3;  // 弱信号
        // 添加到显示
    }
}
```

**玩家体验**：
- ✅ **好处**：雷达显示更清晰
- 🟢 **影响**：几乎无负面影响

### 5. ❌ 电离层/对流层折射（超视距限制）

#### 限制描述
- 射线无法沿大气层弯曲
- 无法实现天波超视距雷达（OTH）
- 无大气波导效应

#### 引擎原因
```c
// 射线是直线传播
// ❌ 无大气分层模型
// ❌ 无电离层反射
// ❌ 无折射率梯度
```

#### 对雷达的影响

**严重程度**：🟡 中等（定位限制）

**缺失的能力**：
```
现实 OTH 雷达：
雷达 ---→ 电离层 ---反射--→ 3000 km 外目标

引擎中：
雷达 ---→ 只能探测视距内 (20-50 km)
```

**雷达定位**：
- ❌ 无法制作**战略级**超视距雷达
- ✅ 可以制作**战术级**视距雷达（20 km 范围）

#### 缓解方案

**方案 1：定位为战术雷达**
```c
// 明确定位：
// - 战术搜索雷达（< 50 km）✅
// - 防空预警雷达（< 100 km）✅
// - 火控雷达（< 20 km）✅
// 
// 不制作：
// - 战略预警雷达（> 1000 km）❌
// - OTH 雷达（> 3000 km）❌
```

**方案 2：高空平台代替**
```c
// 预警机雷达（高度 10 km）
// 地平线距离 = 3.57 * (100 + 3) = 367 km
// 在游戏地图范围内接近"超视距"效果
```

**玩家体验**：
- ✅ **合理定位**：战术雷达足够游戏使用
- 🟢 **地图限制**：游戏地图通常 < 50 km，不需要超视距

### 6. ❌ 表面粗糙度精确建模

#### 限制描述
- 材质系统无粗糙度参数
- 无法区分"抛光钢板" vs "锈蚀钢板"
- 镜面反射 vs 漫反射需手动模拟

#### 引擎原因
```c
// GameMaterial 只提供：
material.GetName();        // "Metal"
material.GetDensity();     // 7850 kg/m³
// ❌ 无 material.GetRoughness()
// ❌ 无 material.GetSurfaceQuality()
```

#### 对雷达的影响

**严重程度**：🟢 轻微

**影响范围**：
- RCS 计算精度略降（5-10% 误差）
- 角度相关 RCS 需手动建模

#### 缓解方案

**方案 1：基于入射角的数学模型**（推荐）
```c
class RDF_ReflectionModel
{
    float CalculateReflectivity(float incidentAngle, MaterialType type)
    {
        float angleRad = incidentAngle * Math.DEG2RAD;
        float cosAngle = Math.Cos(angleRad);
        
        switch (type)
        {
            case MATERIAL_METAL_SMOOTH:
                // 镜面反射为主
                return 0.9 * Math.Pow(cosAngle, 0.5);  // 接近垂直时强反射
                
            case MATERIAL_METAL_ROUGH:
                // 漫反射为主
                return 0.6 * (0.5 + 0.5 * cosAngle);  // 更均匀
                
            case MATERIAL_CONCRETE:
                // 强漫反射
                return 0.3 * (0.8 + 0.2 * cosAngle);
        }
    }
}
```

**方案 2：材质名称映射**
```c
// 从材质名称推断粗糙度
static float EstimateRoughness(string materialName)
{
    if (materialName.Contains("Polish") || materialName.Contains("Chrome"))
        return 0.1;  // 光滑
    else if (materialName.Contains("Rust") || materialName.Contains("Corroded"))
        return 0.8;  // 粗糙
    else
        return 0.5;  // 默认
}
```

**玩家体验**：
- 🟢 **影响极小**：玩家无法察觉 5-10% RCS 误差
- ✅ **可接受**：数学模型补偿足够

### 7. ⚠️ 性能限制（扫描密度 vs 帧率）

#### 限制描述
- 射线追踪有性能开销
- 高密度扫描（360° × 90° × 高频）可能降低帧率
- 多目标穿透模式比单目标模式慢

#### 引擎原因
```c
// 每条射线 = 一次物理查询
for (azimuth = 0; azimuth < 360; azimuth += step)
    for (elevation = 0; elevation < 90; elevation += step)
        TraceMove(...);  // 性能开销

// 示例：
// 360°/5° × 90°/5° = 72 × 18 = 1296 条射线/次扫描
// 如果 10 Hz 刷新 = 12960 条射线/秒
```

#### 对雷达的影响

**严重程度**：⚠️ 需要优化

| 扫描模式 | 射线数/扫描 | 性能 | 适用场景 |
|---------|-----------|------|---------|
| 粗略搜索 | 100-500 | ⭐⭐⭐ 好 | 远程预警 |
| 中等精度 | 500-2000 | ⭐⭐ 中等 | 战术搜索 |
| 高精度 | 2000-5000 | ⭐ 较慢 | 火控雷达 |
| 极高精度 | 5000+ | ⚠️ 慢 | 仅用于特殊模式 |

#### 缓解方案

**方案 1：分帧扫描**
```c
class RDF_FramedScanner
{
    int m_CurrentSector = 0;
    int m_SectorsPerFrame = 4;  // 每帧只扫描 90° 的 1/4
    
    void UpdateFrame()
    {
        float startAngle = m_CurrentSector * 90.0;
        float endAngle = startAngle + 90.0;
        
        ScanSector(startAngle, endAngle);  // 只扫描 1 个扇区
        
        m_CurrentSector = (m_CurrentSector + 1) % m_SectorsPerFrame;
        // 4 帧完成 360° 扫描
    }
}
```

**方案 2：自适应分辨率**
```c
// 远距离用粗分辨率，近距离用细分辨率
float GetAngularStep(float distance)
{
    if (distance > 2000)
        return 10.0;  // 远距离：10° 步进
    else if (distance > 500)
        return 5.0;   // 中距离：5° 步进
    else
        return 2.0;   // 近距离：2° 步进
}
```

**方案 3：异步扫描**
```c
// 使用 AsyncTraceMove 不阻塞主线程
GetGame().GetWorld().AsyncTraceMove(trace, OnTraceFinished);
```

**方案 4：LOD（距离层次）**
```c
// 只对感兴趣区域进行高精度扫描
if (IsTargetNearby(lastKnownPos, 500))
    HighResolutionScan(lastKnownPos);  // 精细扫描
else
    LowResolutionScan();  // 粗略扫描
```

**推荐配置**：
```c
// 平衡配置（60 FPS 友好）
- 扫描范围：360° × 30°（水平 × 垂直）
- 角度步进：5° × 5°
- 射线总数：72 × 6 = 432 条/扫描
- 刷新率：2 Hz（每秒 2 次扫描）
- 每秒射线数：864 条/秒
→ 性能开销：< 5% CPU
```

### 8. ⚠️ 网络同步开销（多人游戏）

#### 限制描述
- 雷达样本数据需要网络同步
- 大量目标时数据量大
- 高刷新率增加带宽消耗

#### 引擎原因
```c
// 每个雷达样本需要同步：
class RDF_RadarSample
{
    IEntity m_Target;      // 4 字节（实体 ID）
    vector m_Position;     // 12 字节
    vector m_Velocity;     // 12 字节
    float m_Distance;      // 4 字节
    float m_RCS;           // 4 字节
    // 总计：~40 字节/样本
}

// 100 个目标 × 40 字节 × 10 Hz = 40 KB/s
```

#### 对雷达的影响

**严重程度**：⚠️ 需要优化

**网络开销估算**：

| 目标数 | 刷新率 | 带宽消耗 | 评估 |
|-------|-------|---------|------|
| 10 | 10 Hz | 4 KB/s | 🟢 可接受 |
| 50 | 10 Hz | 20 KB/s | 🟢 可接受 |
| 100 | 10 Hz | 40 KB/s | 🟡 较高 |
| 100 | 2 Hz | 8 KB/s | 🟢 优化后可接受 |

#### 缓解方案

**方案 1：降低刷新率**
```c
// 雷达不需要 60 Hz 刷新
// 2-5 Hz 足够（真实雷达也是 2-10 秒刷新）
m_UpdateRate = 2.0;  // 2 Hz
```

**方案 2：优先级同步**
```c
class RDF_NetworkOptimizer
{
    void SyncPrioritizedTargets()
    {
        // 只同步重要目标
        foreach (sample : m_Samples)
        {
            float priority = CalculatePriority(sample);
            
            if (priority > 0.5)
                SyncToClients(sample);  // 同步高优先级目标
            // 低优先级目标不同步或降低频率
        }
    }
    
    float CalculatePriority(RDF_RadarSample sample)
    {
        float priority = 0;
        
        // 距离近 = 高优先级
        priority += (1.0 - sample.m_Distance / m_MaxRange) * 0.3;
        
        // 大 RCS = 高优先级
        priority += Math.Clamp(sample.m_RCS / 10.0, 0, 1) * 0.3;
        
        // 高速 = 高优先级
        priority += Math.Clamp(sample.m_Velocity.Length() / 50.0, 0, 1) * 0.4;
        
        return priority;
    }
}
```

**方案 3：增量同步**
```c
// 只同步变化的目标
// 静止目标不重复发送
if (sample.HasChangedSince(lastSyncTime))
    SyncToClients(sample);
```

**方案 4：压缩数据**
```c
// 降低精度减少字节数
class CompressedRadarSample
{
    int m_TargetID;        // 4 字节
    short m_Distance;      // 2 字节（精度 1m，范围 0-65km）
    short m_Azimuth;       // 2 字节（精度 0.01°）
    char m_Elevation;      // 1 字节（精度 1°）
    char m_Speed;          // 1 字节（精度 1 m/s）
    // 总计：10 字节/样本（压缩 75%！）
}
```

## 📊 限制影响总结

### 按严重程度分类

| 严重程度 | 限制数量 | 限制项 |
|---------|---------|--------|
| 🔴 严重 | **0** | 无 |
| 🟡 中等 | **3** | 建筑穿透、地形遮挡、超视距 |
| 🟢 轻微 | **3** | 多路径、表面粗糙度、绕射 |
| ⚠️ 需优化 | **2** | 性能、网络 |

### 按雷达类型影响
